# ADR 0005: ベクターレイヤー(再編集可能なオブジェクト層)

- 状態: 採用(段階導入。Phase 1 = ドメイン基盤 + 描画、Phase 2 = 画面接続 実装済み)
- 日付: 2026-06-15
- 関連: [0004](0004-raster-canvas-engine.md)(ラスターエンジン)

## 決定

ラスターレイヤー(ADR 0004)と**共存**する、新しい**ベクターレイヤー**型を段階導入する。ベクターレイヤーは確定後も無劣化で再編集できる**ベクターオブジェクトの順序付き集合**として保持する。

- **オブジェクト**: ストローク(点列)・図形(`ShapeKind` + 矩形)を `VectorObject`(pure Dart, 不変)として表す。各オブジェクトは色・線幅・座標を持ち、移動 / 色変更 / 線幅変更 / ヒットテストができる。
- **レイヤー**: `VectorLayer` がオブジェクトの並び(描画順)を保持し、追加 / 削除 / 並べ替え / 当たり判定(最前面) / 個別編集を提供する。
- **描画**: `vector_render`(ui 層)が `VectorLayer` を `Canvas` へ描く。ラスター合成へ渡すときは、その場でラスタライズ(`Picture.toImage`)して 1 枚の `ui.Image` として既存の合成パイプラインに載せる。
- **永続化**: ベクターはオブジェクトの JSON として保存できる(無劣化往復)。エクスポート PNG はラスタライズ結果。

## 背景

ユーザー要望「本格ベクターレイヤー(全ストロークをパスとして保持し、無限に再編集・無劣化拡大)」。ADR 0004 のラスターエンジンはピクセル機能(塗りつぶし・フィルタ・ブレンド)に最適だが、確定後の個別オブジェクト編集や無劣化拡大はできない。

| 候補 | 評価 |
| --- | --- |
| ラスターを全面ベクター化 | ibis 系のピクセル機能(塗り・フィルタ)を失う。全面再設計でリスク大 |
| ベクターを別レイヤー型として共存 | ピクセル機能を保ったまま、ベクターの再編集性を足せる。合成境界で吸収 |
| 既存ストロークを裏で常にベクター保持 | 焼込済みラスターとの整合(塗り・フィルタ後)が破綻する |

→ **共存(別レイヤー型)** を採る。

## 理由

- ラスター機能(ADR 0004)を一切壊さず、ベクターの「無劣化・再編集」を付加できる。
- ベクターの幾何・編集ロジックは **pure Dart の domain 層**(`domain/vector/`)に置けるため、`dart:ui` 非依存で決定的に unit テストできる(`docs/architecture.md` の依存ルール、ADR 0003 の方針に合致)。
- 合成は「ベクター → その場で `ui.Image` 化 → 既存の合成へ」とすることで、レイヤースタック / ブレンド / 不透明度の既存実装をそのまま使える。

## 結果(段階導入)

- **Phase 1(本 ADR で実装)**: `domain/vector/`(`VectorObject` = ストローク/図形、`VectorLayer` = 集合と編集操作)+ `ui/canvas/vector_render`(Canvas 描画)。すべて unit / widget テスト付き。まだ描画画面へは接続しない。
- **Phase 2(実装済み)**: ラスター層の上に重なる単一のベクターオーバーレイ(`VectorController`)を導入。`draw_surface` のポインタ処理を接続し、ベクターモードで brush→ストローク / shape→図形 / select→選択・移動を積む。編集バーで削除・色変更・独立 undo/redo。合成・エクスポートはラスタライズ(`RasterPainter` がラスター層の上に描画)。
  - テキストもベクターオブジェクト(`VectorText`)として保持し、ラスター焼き込みをやめた。テキストツールのタップで新規作成/既存テキストの再編集(内容・サイズ・太字・下線・取消線・自由な色)ができる。描画ボックスのサイズは UI 層が `TextPainter` で測って domain へ渡す(domain は `dart:ui` 非依存を維持)。
  - 配置済みオブジェクトを**長押し**すると「調整モード」に入り、1 本指ドラッグで移動・2 本指ピンチで一様拡縮(`VectorObject.scaled`)できる。長押しが何も無い所なら従来どおりツール UI をトグル。選択枠に四隅ハンドルを表示し、調整バー(色/削除/完了)を出す。
  - 次の発展: ベクターを per-layer のレイヤー種別へ昇格(レイヤーパネル統合)、ノード(頂点)編集。
- **Phase 3**: ベクターの JSON 永続化(`GalleryStore` 拡張 or 併存ファイル)、ノード(頂点)編集、ベクター/ラスターの相互変換。
- 当面、エクスポート PNG はラスタライズ結果(ピクセル)。SVG 等のベクター書き出しは別途検討する。
- 性能(オブジェクト数が多い層の毎フレーム描画)が問題化したら、確定オブジェクトのキャッシュ画像化を別 ADR で検討する。
