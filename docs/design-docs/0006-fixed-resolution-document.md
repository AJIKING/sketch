# ADR 0006: 固定解像度ドキュメント(キャンバスサイズの選択)

- 状態: 採用(段階導入。Phase 1 = サイズ選択 + フィット表示 実装済み)
- 日付: 2026-06-15
- 関連: [0004](0004-raster-canvas-engine.md)(ラスターエンジン)

## 決定

キャンバスを「画面サイズ追従」だけでなく、**新規作成時に選んだ固定解像度のドキュメント**として持てるようにする。固定ドキュメントは表示域へ**中央フィット**(アスペクト比保持・レターボックス)で表示し、ピンチで拡大縮小・パンできる。

- `CanvasScreen` は任意の `documentSize`(null = 画面サイズ追従の従来挙動)を受け取る。
- `DrawSurface` は `documentSize` があればそれを `_docSize` に固定し、`ViewportTransform.fit` で表示域へ収める(表示域変化のたび再フィット)。null のときは従来どおり画面サイズに追従(回転で全面)。
- 新規作成フローでサイズを選ぶ(プリセット:画面サイズ / 1080² / 2048² / 1080×1920 / 1920×1080)。
- 既存スケッチを開くときは、保存 PNG の解像度を `documentSize` として復元する(保存時の見た目を保つ)。

## 背景

ibisPaint / Adobe 等は「印刷品質・トレース・SNS 規定サイズ」のために任意解像度のキャンバスを前提とする。本アプリは画面サイズ=キャンバスだったため、高解像度出力や固定比率の制作ができなかった。一方で「横にしたら横全面」(ADR なしの UX 改善)も活かしたい。

| 候補 | 評価 |
| --- | --- |
| 画面サイズのみ(現状) | 実装は単純だが高解像度・固定比率が不可 |
| 固定解像度のみ | 印刷品質は得られるが、手軽な“全面スケッチ”の使い心地を失う |
| 両対応(`documentSize` null で画面サイズ、指定で固定) | 既存 UX を残しつつ制作用途を満たす |

→ **両対応**を採る。

## 理由

- `documentSize` の有無で分岐するだけで、既存の画面サイズ挙動(回転で全面・ズーム保持)を壊さずに固定解像度を足せる。
- 表示は `ViewportTransform.fit`(中央フィット)で吸収し、合成・焼き込み・エクスポートは従来どおり `_docSize` 基準。ピクセル処理(ADR 0004)に変更不要。

## 結果(段階導入)

- **Phase 1(本 ADR で実装)**: サイズ選択 + 固定ドキュメントのフィット表示 + 既存 PNG 解像度での復元。
- **性能の注意**: 大きいキャンバス(例 2048²)は、確定毎に全レイヤーを `ui.Image` 化する現エンジンではメモリ/負荷が大きい(履歴 16 × レイヤー枚数分の画像保持)。**タイル化・差分履歴**は別 ADR で対応する。当面はプリセット上限と履歴上限で実用域に収める。
- **Phase 2(次フェーズ)**: 作成後のキャンバスリサイズ/切り抜き、向き変更、DPI/テンプレート、レイヤー別書き出し。
