# ADR 0008: テキストの多書体(google_fonts)とオフライン方針の緩和

- 状態: 採用
- 日付: 2026-06-15
- 関連: [0005](0005-vector-layers.md)(ベクターレイヤー=テキスト)

## 決定

テキスト(`VectorText`)に**フォント選択(十数種)**と**2 色グラデーション**を追加する。多書体は `google_fonts` パッケージで導入する。

- `VectorText` に `fontFamily`(空 = 既定)、`gradient`、`secondColorHex` を追加(pure Dart のデータ)。
- 描画(`buildVectorTextPainter`)は `fontFamily` を `GoogleFonts.getFont` で解決し、`gradient` のときは前景に横方向 `ui.Gradient.linear` シェーダを掛ける。
- フォント一覧は `lib/src/ui/canvas/text_fonts.dart`(日本語が描けるファミリ中心に十数種)。
- Android は `INTERNET` 権限を追加(初回取得のため)。iOS は HTTPS 既定許可で追加設定不要。

## オフライン方針との関係(重要)

本アプリの原則は「オフライン完結」(CLAUDE.md)。`google_fonts` は**初回表示時に各フォントをネットワーク取得**し、以降は端末にキャッシュする。したがって厳密には「初回のみオンライン必須、以降はオフライン可」へ**緩和**される。これはユーザー(プロダクトオーナー)が「十数種のフォントを用意したい」要件に対し、調達方法として google_fonts を選択した結果である。

| 候補 | 評価 |
| --- | --- |
| .ttf を assets に同梱 | 完全オフライン維持。ただしフォント本体 + 各ライセンス(OFL 等)をリポジトリに同梱する必要があり、容量も増える |
| **google_fonts(初回のみ取得)** | **採用**。実装が最小で多数の書体を即利用可。初回だけ通信(以降キャッシュ)= オフライン原則を一部緩和 |
| 端末システムフォント中心 | 通信不要だが、確実に揃う種類が少なく iOS/Android で見た目が割れる |

## ボックス測定とフォントのロード順

テキストの当たり判定/選択枠に使う `boxWidth/boxHeight` は確定時に `TextPainter` で実測する。google_fonts はフォントを非同期ロードするため、未ロードの寸法(フォールバック)で確定すると、後で実フォントが届いたとき字形と box がずれる。これを避けるため、**確定前に対象フォントのロード完了を待ってから測る**(`ensureFontLoaded`)。描画ホットパスでは毎フレームの `getFont` を避け、解決済みファミリ名をキャッシュする(`_resolveFontFamily`)。ロード future の失敗は握り潰し、未処理例外/ログ汚染を残さない。

## 限界 / 今後

- 初回オフライン時は取得に失敗し、該当フォントはフォールバック(標準)で描画される(クラッシュはしない。テストは `GoogleFonts.config.allowRuntimeFetching = false` で通信せず検証)。この場合 box もフォールバック寸法で確定し、後でオンラインになって実フォントが届くと表示と box がわずかにずれる(同一セッション内の限定ケース)。
- 完全オフラインが要件化したら、`.ttf` 同梱(上表の代替案)へ移行する。`fontFamily` はデータとして残るので移行時も互換。
- グラデーションは横方向の 2 色線形のみ(縦/斜め/多色は今後)。下線/取消線の色は始点色固定(グラデにしない)。
