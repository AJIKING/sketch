# ADR 0007: レイヤーマスク(非破壊の可視範囲制御)

- 状態: 採用(Phase 1 = 追加 / 解除 / 編集 / 合成 / undo 実装済み)
- 日付: 2026-06-15
- 関連: [0004](0004-raster-canvas-engine.md)(ラスターエンジン)、[0003](0003-drawing-determinism.md)(undo/redo)

## 決定

レイヤーごとに任意の**マスク**を持てるようにする。マスクはレイヤー本体の画素を一切変えずに、**アルファで可視範囲を制御**する非破壊の仕組み。

- マスクの有無は `LayerMeta.hasMask`(構成の一部)。
- マスクの画素はレイヤー画素と同じ `CanvasSurface` に、**派生 id** `maskLayerId(layerId)`(= `'$layerId#mask'`)をキーに保持する。新しい画像レイヤーを増やさず、既存の snapshot/restore 機構をそのまま使える。
- 合成(`RasterPainter._drawContent`)は、`hasMask` のとき本体画素を `saveLayer` で包み、最後にマスク画像を `BlendMode.dstIn` で重ねて切り抜く(マスクの alpha>0 の箇所だけ本体が見える)。ライブプレビューにはマスクを掛けない。
- 初期マスクは**全面不透明(白)** = どこも隠さない。**消しゴムで消すほど隠れ、ブラシで塗ると戻る**(アルファマスク)。
- 編集は「マスク編集モード」(`CanvasController.maskEditing`)。ON かつアクティブにマスクがある間、ラスター描画(ストローク / 図形 / グラデ / 選択内消去・塗り)の焼き込み先を `drawTargetId` がマスク id へ振り替える。バケツ塗り・スポイト・自動選択は本体のまま。

## undo/redo

マスクの追加・解除は**構成変更**なので [ADR 0003] の構造対応履歴(`StackEdit`)で巻き戻す。`StackEdit` は全レイヤーの本体画素に加えて**各マスク画素も**スナップショットするため、フラグ(構成)と画素を一括で復元できる。マスクへのストロークは通常の `PixelEdit`(対象 = マスク id)で個別に undo できる。

## 背景

ibisPaint / Adobe のマスクは制作の要。本アプリはラスター=`ui.Image` 1 枚/レイヤー(ADR 0004)なので、本体を直接消す破壊編集しか無かった。マスクを別 id の画像 + `dstIn` 合成として足すことで、エンジン構造を変えずに非破壊の可視制御を実現する。

| 候補 | 評価 |
| --- | --- |
| 輝度マスク(白=表示/黒=非表示) | Photoshop 互換だが、輝度→alpha 変換が要る。MVP では複雑 |
| **アルファマスク(dstIn)** | **採用**。実装が単純で既存の焼き込み/合成にそのまま乗る |
| マスク専用の別ストア | 不要な複雑化。派生 id で同一ストアに同居できる |

## 限界 / 今後

- 下レイヤーが非通常ブレンドのときの結合と同様、ライブプレビューはマスク未適用(確定時に反映)。
- 輝度ベースのマスク、マスクのみ表示、レイヤー↔マスクの反転/適用は今後。
