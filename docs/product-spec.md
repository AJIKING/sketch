# Hatch — プロダクト仕様(プロトタイプ準拠)

この文書は、HTML プロトタイプ `docs/prototype/hatch-sketch-app.html` から抽出した、Flutter 本実装の対象仕様。プロトタイプと本仕様が食い違う場合はプロトタイプを確認して本仕様を更新する。

## コンセプト

スマホで完結する軽量スケッチ/お絵描きアプリ「Hatch(ポケットのアトリエ)」。紙(paper)とインク(限られたアーティストパレット)をモチーフにした、暖色の暗い「アトリエ」UI。レイヤー・複数ブラシ・カラーピッカーを備える。オフライン完結で、バックエンド・認証は持たない。

デザイントークン(色・角丸・影・イージング)はプロトタイプの `:root` 変数に準拠する。基調色: shell `#1c1813` / surface `#221d17` / paper `#efe7d6` / ink `#f3ecdd`。アクセント色は「現在のインク色」で動的に変わる。

## 画面構成

2 ビュー + 複数のボトムシート。

1. **ギャラリー**(`galleryView`)
2. **キャンバス**(`canvasView`)
   - ボトムシート: カラー / ブラシ / レイヤー / メニュー

ギャラリー → キャンバスはスライドアップ遷移(キャンバスが下から せり上がり、ギャラリーは縮小フェード)。

### ギャラリー

- ヘッダ: ブランド表記「Hatch / Pocket Atelier」、右上にスケッチ点数(例「3点のスケッチ」)。
- グリッド: 先頭に「新規キャンバス」ボタン(＋)、続いて保存済みスケッチのサムネイル(各カードに画像 + メタ「タイトル · 日時」)。
  - プロトタイプは手続き的に生成したサンプル 3 点(「朝の習作」「街の輪郭」「余白の練習」)を表示する。本実装では永続化したユーザーのスケッチを表示する(ADR 0001)。
- 操作:
  - 「新規キャンバス」→ まっさらなキャンバスを開く(全レイヤークリア、履歴リセット)。
  - サムネイルタップ → そのスケッチをキャンバスで開く。
- 点数表示はスケッチ保存のたびに更新する。

### キャンバス

紙の上に複数レイヤーを合成して表示する描画画面。固定 UI:

- **トップバー**: 戻る(中断してギャラリーへ。現在の内容を保存)、取り消し(undo)、やり直し(redo)、メニュー。undo/redo は対象スタックが空のとき disabled。
- **左レール(縦スライダー)**:
  - SIZE: ブラシサイズ 1–80(既定 14)。
  - OPAC: 不透明度 0–100%(既定 100%)。
  - スライダーは縦方向ドラッグ + キーボード(↑↓←→)操作に対応し、現在値をバブル表示する。
- **ボトムドック**: ブラシ / スマッジ / 消しゴムの 3 ツール、区切り、レイヤーボタン(バッジ = レイヤー数)、カラーボタン(現在色を表示)。
- ツール選択中のブラシツールを**もう一度タップ**するとブラシライブラリ(ブラシシート)が開く。

#### ボトムシート

scrim(暗幕)タップで閉じる。同時に 1 つだけ開く。

- **カラーシート**:
  - SV(彩度×明度)正方形 + Hue 帯の HSV ピッカー。
  - HEX 値表示と現在色チップ。
  - Studio Palette: 固定 10 色(`#CF4A2C` 朱 / `#E08A2E` / `#C69749` 黄土 / `#5E8A4E` / `#2C4A63` 藍 / `#46357A` / `#A23B68` / `#2A2620` 墨 / `#7A726A` / `#EFE7D6` 紙)。
  - Recent: 直近使用色を最大 8 件(重複は先頭へ寄せる)。
  - 色を変えると UI のアクセント色(`--accent`)とカラーボタンが追従する。
- **ブラシシート**: 4 種のブラシをプレビュー付きで一覧。選択中にチェック。選択するとツールはブラシに戻る。
  - インク(ink): なめらかで均一。速度で太さが変化。
  - ペンシル(pencil): ざらりとした鉛筆。重ねるほど濃く。
  - マーカー(marker): 平らで半透明。重なりが色を作る。
  - エアブラシ(air): やわらかく霧状に積もる。
- **レイヤーシート**: レイヤーを上から順に一覧。各行にサムネイル、名前、不透明度 %、表示トグル(目)、削除。「追加」ボタンで新規レイヤー。行(情報部)タップでアクティブレイヤー切替。
  - 初期状態: レイヤー 2 枚(「レイヤー 1」「レイヤー 2」)、上のレイヤーをアクティブ。
  - 最後の 1 枚は削除不可(トースト「最後のレイヤーは消せません」)。
  - 非表示レイヤーには描けない(トースト「非表示のレイヤーには描けません」)。
- **メニューシート**:
  - 「画像として保存」: 合成結果を PNG としてエクスポート(プロトタイプはダウンロード。本実装は端末への保存 = `ImageExporter`)。
  - 「完了してギャラリーへ」: 現在の内容を保存してギャラリーへ戻る。
  - 「このレイヤーを消去」(danger): アクティブレイヤーをクリア(undo 可能)。

## 描画仕様

### キャンバスとレイヤー

- 表示キャンバスは要素サイズ × DPR(`min(devicePixelRatio, 2)`)で確保する。
- 各レイヤーは同サイズのオフスクリーン。`visible` と `opacity` を持つ。
- 合成(composite): 紙色(`#efe7d6`)で塗りつぶし → 表示レイヤーを下から順に、各レイヤーの不透明度で重ねる。
- 描画はアクティブレイヤーに対して行う。

### ツール

| ツール | 合成方法 | 振る舞い |
| --- | --- | --- |
| ブラシ | source-over | 選択中のブラシプリセット × 現在色 × サイズ × 不透明度で描く |
| スマッジ | source-over | ストローク開始点の色をサンプリングし、低 α でやわらかく引き延ばす |
| 消しゴム | destination-out | 不透明度に応じてアクティブレイヤーを消す |

### ブラシプリセット

各プリセットは flow(基本 α)/ soft(ぼかし)/ scatter(散り)/ spacing(間隔)を持つ。

| key | flow | soft | scatter | spacing | 特徴 |
| --- | --- | --- | --- | --- | --- |
| ink | 1.0 | 0 | 0 | 0.25 | 線。速度が速いほど細くなる(`w = base × clamp(1.15 − speed×0.9, 0.4, 1)`) |
| pencil | 0.5 | 0 | 0.9 | 0.5 | 粒状の点を散らして重ねる |
| marker | 0.55 | 0.1 | 0 | 0.2 | 平筆の半透明線。重なりで濃くなる |
| air | 0.18 | 1.0 | 0.2 | 0.18 | 柔らかい放射グラデーションを積む |

- ストロークは 2 点間を補間してスタンプする。ink / marker は線分描画、pencil / air は spacing 間隔でダブを置き、scatter 分だけ位置を散らす。
- 散らし(scatter)・粒状感の乱数は、本実装では注入した `Random(seed)` を使う(プロトタイプの `Math.random()` 直叩きは踏襲しない)。これにより描画結果を決定的に再現できる。
- 筆致の速度はストローク中の移動距離 / 経過時間から求める。本実装では時間源を `Clock` で差し替え可能にする。

### Undo / Redo

- ストローク開始時にアクティブレイヤーのスナップショットを undo スタックへ積む(最大 16 件、超過は古いものを破棄)。
- 新規ストロークで redo スタックをクリアする。
- 「このレイヤーを消去」も undo 対象。
- undo/redo はレイヤー単位(どのレイヤーのスナップショットかを保持して復元する)。

## 状態・データ

### キャンバス状態(セッション)

- アクティブツール、選択ブラシ、サイズ、不透明度、現在色(HSV)、最近色。
- レイヤー配列(名前 / 画素 / 表示 / 不透明度)、アクティブレイヤー index。
- undo / redo スタック。

### 永続化(本実装で追加)

プロトタイプはリロードで状態が消える(in-memory のみ)。本実装ではギャラリーを永続化する。保存方式(レイヤーを保持するか、合成 PNG のみか)は ADR 0001 で決定する。

- スケッチ: サムネイル / 作成・更新日時 / タイトル(任意)。
- 最近使った色は永続化してもよい(任意)。

## 採点・通信

- なし(描画アプリのため)。バックエンド・認証・通信を持たない。

## 非機能・制約

- 単一 locale(ja)。
- light / dark の概念はなく、アトリエ(暗色)テーマ固定。
- 入力はポインタイベント(タッチ / スタイラス)。筆圧は初期 scope では未使用(将来拡張)。
- アクセシビリティ: 主要ボタン・スライダー・スウォッチに semantics ラベルを付ける(プロトタイプの `aria-label` 相当)。

## プロトタイプとの差分方針

- 乱数は seed 注入で決定化する(プロトタイプは `Math.random()`)。
- 時間源は `Clock` 注入(プロトタイプは `performance.now()`)。
- 永続化を追加する(プロトタイプは in-memory)。
- 画像エクスポートは端末保存 / 共有に置き換える(プロトタイプはブラウザダウンロード)。
- レイヤー合成・ブラシ・カラー変換のロジックは pure Dart の domain 層へ寄せ、UI から分離してテスト可能にする(`docs/architecture.md`)。
